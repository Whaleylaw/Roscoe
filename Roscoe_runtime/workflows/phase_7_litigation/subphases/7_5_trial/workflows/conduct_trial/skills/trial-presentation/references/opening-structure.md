# Opening Statement Structure Reference

## Purpose of Opening

- Tell jury what case is about
- Preview the evidence
- Establish theme
- Build rapport

**NOT argument** - Save persuasion for closing.

---

## Standard PI Opening Structure

### 1. Introduction (1-2 minutes)

```
Good morning. My name is [Name], and I represent [Client].

This case is about [one sentence summary - the theme].
```

### 2. The Story (5-10 minutes)

Tell what happened:
- Before the accident
- The accident itself
- Immediate aftermath
- Injuries discovered
- Treatment journey
- Current condition

**Use present tense for vividness:**
```
"It's [date]. [Client] is driving home from work..."
```

### 3. The Defendant's Fault (2-3 minutes)

Explain what defendant did wrong:
- Specific negligent acts
- Rules violated
- Why it matters

```
"The evidence will show that the defendant [specific conduct]..."
```

### 4. The Injuries (3-5 minutes)

Describe the harm:
- Injuries sustained
- Treatment required
- Ongoing effects
- Impact on life

**Be specific:**
```
"[Client] suffered [specific injuries]. You will hear from 
Dr. [Name] who performed [surgery]..."
```

### 5. The Damages (2-3 minutes)

Preview the numbers:
- Medical expenses
- Lost wages
- Pain and suffering

```
"The medical bills total $[X]. You'll see every bill, 
every record..."
```

### 6. Conclusion (1 minute)

Roadmap and ask:
```
"At the end of this trial, after you've heard all the 
evidence, I'm going to ask you to return a verdict that 
fully compensates [Client] for all the harm caused by 
the defendant's negligence."
```

---

## Themes

### Strong Themes

- **Responsibility**: "This case is about taking responsibility"
- **Safety**: "Rules exist for a reason"
- **Impact**: "How one moment changed everything"
- **Truth**: "The evidence speaks for itself"

### Theme Integration

Weave theme throughout:
- Opening: State theme
- Direct: Reinforce theme
- Cross: Challenge theme violations
- Closing: Return to theme

---

## Do's and Don'ts

### Do:
- Tell a story
- Use plain language
- Make eye contact
- Reference specific evidence
- Be confident

### Don't:
- Argue
- Overstate evidence
- Make promises you can't keep
- Read from notes
- Be boring
