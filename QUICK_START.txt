LEGAL COMPLIANCE CHECKLIST
================================================================================

Before deploying any scraping:

__ 1. Data is publicly available (no stolen login credentials)
__ 2. Using legitimate credentials only
__ 3. Checked website's robots.txt and Terms of Service
__ 4. No CAPTCHA solving on protected systems
__ 5. No proxy rotation to hide identity
__ 6. Documented purpose and authorization
__ 7. Legal approval obtained for sensitive data
__ 8. Audit logs will be maintained
__ 9. Rate limiting will be respected
__10. No circumventing technical protections
__11. For PACER: Using official API or firm credentials
__12. For LexisNexis/Westlaw: Checked if automation permitted

RECOMMENDATIONS AT A GLANCE
================================================================================

DO:
  + Use Playwright + AgentQL for most paralegal tasks
  + Use legitimate credentials for authenticated systems
  + Document everything for legal compliance
  + Get legal approval for sensitive data
  + Maintain detailed audit trails
  + Respect robots.txt and rate limiting
  + Use official APIs when available (PACER, SEC, etc.)

DON'T:
  - Use tools emphasizing bot evasion (Oxylabs, BrightData Scraper)
  - Solve CAPTCHAs on protected systems
  - Rotate proxies to hide your identity
  - Circumvent access controls you're not authorized for
  - Create fake accounts to access data
  - Share unencrypted medical data
  - Republish copyrighted content
  - Use Oxylabs or BrightData Scraper (legal risk > benefit)

LEGAL FRAMEWORK SUMMARY
================================================================================

PUBLIC DATA (no login required)
  Status: GENERALLY LEGAL
  Rules: Respect robots.txt, no bot evasion
  Tools: Any Tier 1 tool

AUTHENTICATED DATA (behind login)
  Status: LEGAL if using YOUR credentials
  Rules: Don't create fake accounts, respect ToS
  Tools: Playwright, AgentQL, Requests

CIRCUMVENTING PROTECTIONS
  Status: RISKY (CFAA/DMCA violation)
  Avoid: CAPTCHA solving, proxy rotation, IP hiding
  Risk: Criminal liability

COPYRIGHT & FAIR USE
  Status: Scraping legal; usage determines legality
  Safe: Research, internal use, case preparation
  Risky: Republishing content, commercial use

PRIVACY LAWS
  Compliance: HIPAA (medical), GDPR (EU), CCPA (CA)
  Requirement: Secure handling of personal data
